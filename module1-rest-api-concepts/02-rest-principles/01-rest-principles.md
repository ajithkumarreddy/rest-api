# REST Principles

REST is not just about using HTTP methods like GET or POST. An API is considered RESTful when it follows a set of architectural constraints described by Roy Fielding.

The six main REST principles are:
1. Client-Server Architecture
2. Statelessness
3. Cacheability
4. Uniform Interface
5. Layered System
6. Code on Demand (optional)

These principles help make APIs simple, scalable, and easy to maintain.
