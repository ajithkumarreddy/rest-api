# Client-Server Architecture

In a client-server architecture, the client and server are separate parts of the system.

- The client is responsible for sending requests and presenting data to the user.
- The server handles business logic, storage, authentication, and data processing.

This separation allows each side to evolve independently. It also improves scalability because the server can be updated or expanded without changing the client in the same way.

In REST APIs, this architecture is a core principle because it keeps the API simple and modular.
