Example URL:
https://api.example.com:3000/users/12?sort=name#section=headers

Parts of a URL:
- Protocol (http/https): tells the browser how to communicate with the server.
- Hostname: identifies the server, such as api.example.com.
- Subdomain: a prefix before the main domain, such as api.
- Domain: the main site name, such as example.com.
- Port: optional value that identifies a specific application on the server, such as :3000.
- Path: identifies the resource being requested, such as /users/12.
- Route parameter: a dynamic value inside the path, such as 12.
- Query string: extra options or filters sent after ?, such as ?sort=name.
- Fragment: a browser-only section identifier, such as #section=headers, and it is not sent to the server.
