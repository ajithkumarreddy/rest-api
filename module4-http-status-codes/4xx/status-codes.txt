4xx - Client error responses

400 - Bad Request
The request is malformed or missing required data.
Example: wrong format, missing field, invalid JSON.

401 - Unauthorized
The client has not provided valid authentication.
Example: no token was sent.

403 - Forbidden
The client is authenticated, but does not have permission to access the resource.
Example: trying to delete an admin-only user.

404 - Not Found
The requested resource does not exist.
Example: GET /users/9999

409 - Conflict
The request conflicts with the current state of the server.
Example: trying to create a user that already exists.

422 - Unprocessable Entity
The request is valid syntactically, but fails business rules.
Example: valid JSON but invalid action.
